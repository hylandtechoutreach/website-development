# Website Development: 290160
_Materials and lesson plans for the Website Development course offered by Hyland Tech Outreach_

## Course Overview
This course covers the basics of web development, introducing static  HTML webpages, essential CSS styles, and interactive JavaScript. By the end of the course, students will be able to create and share their own dynamic, interactive, and compelling websites based on their own personal interests. They will also gain a comprehensive understanding of fundamental programming concepts that apply to computer science beyond simply web development.

## Delivery Model Description
Students join us in our classrooms (Westlake and Cleveland Midtown) and utilize our industry-standard equipment to learn in a welcoming yet rigorous environment.

## Scope & Sequence: Pacing Guide
The course is split up into two units, each meant to comprise one semester. Lessons will be presented in order, as they build on each other. The ultimate goal is for students to develop the skills they need to create a fully-functioning interactive website. Each lesson brings them closer to realizing this goal.

There are 18 total lessons, and each lesson is designed to be two hours long.

### Unit One: HTML & CSS
The first unit covers the basics of HTML & CSS, including a range of HTML tags, essential CSS properties and selectors, responsive design, and visual possibilities.

- [1.01: Introduction to Website Development](1.01IntroductionToWebsiteDevelopment/README.md)
- [1.02: Introduction to HTML Elements](1.02IntroductionToHtmlElements/README.md)
- [1.03: More HTML Elements](1.03MoreHtmlElements/README.md)
- [1.04: HTML Review](1.04HtmlReview/README.md)
- [1.05: Introduction to CSS](1.05IntroductionToCss/README.md)
- [1.06: CSS with Divs and Spans](1.06CssWithDivsAndSpans/README.md)
- [1.07: Responsive Design](1.07ResponsiveDesign/README.md)
- [1.08: Transitions & Transforms](1.08TransitionsAndTransforms/README.md)
- [1.09: Final HTML & CSS Project](1.09HtmlCssProject/README.md)

At the end of the unit, students will present their personal projects built using HTML & CSS. The final project will be used as the primary assessment for the successful completion of the unit. The unit emphasizes the practical application of HTML & CSS techniques as opposed to comprehensive review of language features.

### Unit Two: JavaScript
The second unit covers interactivity and fundamental programming concepts with JavaScript, including data types, buttons and functions, `if`/`else` program flow, working with the Document Object Model API, and for loops. Each lesson is designed to be two hours long.

- [2.01: HTML & CSS Review](2.01HtmlCssReview/README.md)
- [2.02: Introduction to JavaScript](2.02IntroductionToJavaScript/README.md)
- [2.03: Buttons & Functions](2.03ButtonsAndFunctions/README.md)
- [2.04: Conditionals](2.04Conditionals/README.md)
- [2.05: JavaScript Review](2.05JavaScriptReview/README.md)
- [2.06: DOM Manipulation](2.06DomManipulation/README.md)
- [2.07: DOM Manipulation Continued](2.07DomManipulationContinued/README.md)
- [2.08: For Loops](2.08ForLoops/README.md)
- [2.09: Final Projects](2.09FinalProjects/README.md)

At the end of the second unit, students will present their final projects for the course. These final projects will be used as the final assessment for course completion. The functional use of JavaScript to bring ideas to life is the primary focus of the unit, and it does not venture into the more technical language details.

## Differentiation
Students taking this course may have a variety of learning needs - it is important to provide a good experience for _all_ students regardless of differences.

### Out-of-Classroom Learning
Students are welcome to complete each lesson on their own - materials including presentations, code-along activity guides, and other resources are provided. Guidance and assistance is available asynchronously from instructors via email during business hours.

### Pair Programming
If a student is struggling to quickly grasp concepts (or if they are grasping them too quickly), [pair programming](https://en.wikipedia.org/wiki/Pair_programming) can ensure focus and comprehension, without directly requiring an instructor to neglect the other students while spending time with one individual. [Pair programming has proven to aid in code quality as well as shared understanding of code.](https://hci.stanford.edu/publications/2005/pairs/PairProgramming-WhenWhy.pdf)
